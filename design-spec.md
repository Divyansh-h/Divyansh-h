# GitHub Profile README Design Specification

## 1. Color Palette (Monochrome)
A sophisticated, hand-crafted grayscale palette prioritizing text clarity and native GitHub integration.

*   **Background:** `#0d1117` (Native GitHub Dark)
*   **Primary Text:** `#c9d1d9` (Native GitHub Light Gray)
*   **Muted Text / Borders:** `#30363d` (Native GitHub Border)
*   **Accent (High Contrast):** `#ffffff` (Pure White)

## 2. Typography Approach
Clean, readable, and entirely reliant on native Markdown rendering.

*   **Primary Font Style:** Native GitHub Markdown defaults. Avoid injecting external fonts or SVG-rendered text (like typing animations).
*   **Structure:** Rely on standard `<h1>`, `<h2>`, and `<h3>` tags for structural hierarchy instead of image-based banners.

## 3. Layout Grid
Minimalist structure with a focus on breathing room.

*   **Alignment:** Left-aligned or centered text, heavily favoring standard paragraph blocks over cluttered flexbox grids.
*   **Whitespace:** Use clean markdown dividers (`---`) and standard spacing to separate sections logically.

## 4. Component Consistency Rule
All interactive or graphical elements must feel like native, utilitarian developer tools.

*   **Badges:** Use `flat-square` or standard flat style. Colors must be strictly monochrome (Background `#0d1117` or `#161b22`, Logo `#ffffff` or `#c9d1d9`).
*   **Visual Assets:** Remove automated widgets (like trophies or animated headers). Any stats cards must use a transparent/black background with white/gray text and gray borders.
